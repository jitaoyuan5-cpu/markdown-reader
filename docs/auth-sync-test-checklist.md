# 登录 + 用户数据后台持久化 测试清单

## 1. 测试目标

- 验证登录门禁、账户密码+验证码流程可用且错误提示清晰。
- 验证登录态管理（含过期/退出）行为正确。
- 验证文件与书签与用户绑定并可后台持久化。
- 验证关键安全基线（JWT 密钥配置、验证码一次性、token 存储策略）满足最小要求。

## 2. 执行前准备

1. 配置后端环境变量：`server/.env`（参考 `server/.env.example`）。
2. MySQL 建库并执行迁移：`server/migrations/001_init.sql`。
3. 安装依赖：
   - 根目录：`npm install`
   - 后端：`npm --prefix server install`
4. 启动后端：`npm run dev:server`
5. 启动应用：`npm run dev`

## 4. 手工测试清单（关键路径）

### 4.1 登录流程

1. 未登录启动应用仅显示登录页。
2. 空账户提交：提示“账户不能为空”。
3. 空密码提交：提示“密码不能为空”。
4. 空验证码提交：提示“验证码不能为空”。
5. 刷新验证码后图片变化且可继续提交。
6. 验证码错误：提示“验证码错误或已失效”。
7. 首次新账户登录：自动建号并进入主流程。
8. 已有账户正确密码登录成功。
9. 已有账户错误密码：提示“账户或密码错误”。
10. 退出登录后回到登录页。

### 4.2 登录态与会话

1. 重启应用后会话可恢复（有 refresh token 时）。
2. 强制使 access token 过期后，业务请求触发自动 refresh 成功。
3. refresh token 失效后，自动回登录页（收到会话过期）。

### 4.3 用户数据持久化

1. 登录后自动拉取云端文件列表并可打开。
2. 新建/打开文件后触发云端 upsert，刷新后仍存在。
3. 新增书签后可在云端读取回来。
4. 修改书签备注后可覆盖更新。
5. 删除书签后云端记录消失。

### 4.4 用户隔离

1. 用户 A 创建文件/书签后退出。
2. 用户 B 登录时看不到 A 的文件/书签。
3. 用户 A 再登录，能看到自己的历史数据。

### 4.5 失败与兼容性

1. 后端断开时，文件/书签操作提示“云端同步失败，已保留本地数据”。
2. 浏览器非 Electron 环境不应绕过登录门禁（应停留在未登录态提示）。
3. 最近文件中 `cloud://` 路径可重新打开（有云端数据时）。

## 5. 通过标准

- 自动化验证全部通过。
- 手工关键路径（4.1 ~ 4.4）全部通过。
- 失败路径（4.5）行为符合预期。
